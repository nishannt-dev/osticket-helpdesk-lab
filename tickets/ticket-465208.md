## Issue Reported

---

## Internal Notes (Agent: Nishant)

Received urgent ticket from Maria Hill reporting inability to access the shared network drive. Steps taken:

1. Confirmed Maria's internet and general network connectivity was working -- browsing and email were fine
2. Checked the file server status -- server was online, no outages reported
3. Opened File Explorer on Maria's workstation remotely -- network drive letter was present but showing a red X (disconnected)
4. Attempted to reconnect by clicking the drive -- returned error "The network path was not found"
5. Checked Maria's account permissions in the admin panel -- permissions were fully intact, no changes had been made
6. Reviewed the drive mapping -- confirmed the mapping had been lost after an unexpected restart the previous evening
7. Manually remapped the network drive using the correct UNC path
8. Drive reconnected immediately and all files were accessible
9. Configured the drive mapping to reconnect automatically at login using a logon script
10. Tested by simulating a logout and login -- drive reconnected automatically as expected

Root cause: The network drive mapping was lost after an unplanned system restart the previous evening. The mapping was not configured to persist across restarts.

---

## Resolution Reply (Agent: Nishant)

> Hi Maria, the shared network drive mapping was lost after your computer restarted unexpectedly last evening. I have remapped the drive and configured it to reconnect automatically every time you log in, so this should not happen again. All your files should be accessible now. Let us know if you have any trouble reaching them.
